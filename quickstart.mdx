This guide walks you from zero to joining a Google Meet call as a real-time AI avatar using the `pikastream-video-meeting` skill.

## Prerequisites

- An AI coding agent (Claude Code, OpenClaw, or similar)
- Python 3.10+
- A Google Meet or Zoom link to join

<Steps>
  <Step title="Get a Pika Developer Key">
    Go to [pika.me/dev](https://www.pika.me/dev/) and sign up for a Developer Key. Your key starts with `dk_` and is used to authenticate all API calls made by your skills.

    <Note>
      Keep your Developer Key secure. Treat it like a password — don't commit it to version control.
    </Note>
  </Step>

  <Step title="Set the PIKA_DEV_KEY environment variable">
    Export your key in your terminal session:

    ```bash
    export PIKA_DEV_KEY="dk_your-key-here"
    ```

    <Tip>
      To persist your key across sessions, add the export line to your shell profile (`~/.bashrc`, `~/.zshrc`, or `~/.profile`) so you don't have to re-export it every time.
    </Tip>
  </Step>

  <Step title="Clone the repository and install the skill">
    Clone the Pika Skills repository:

    ```bash
    git clone https://github.com/Pika-Labs/Pika-Skills.git
    ```

    Install the skill's Python dependencies:

    ```bash
    pip install -r Pika-Skills/pikastream-video-meeting/requirements.txt
    ```

    Then tell your agent to install the skill by pointing it at the skill directory:

    ```
    install /path/to/Pika-Skills/pikastream-video-meeting/
    ```

    The agent reads `SKILL.md` and is ready to use the skill — no additional configuration needed.
  </Step>

  <Step title="Set up your avatar (first time only)">
    The first time you use the skill, the agent checks whether `identity/videomeeting-avatar.png` exists. If it doesn't, it will ask:

    > I need an avatar image for the video meeting (a headshot or portrait). Send me an image, or say "generate" and I'll create one for you.

    You have two options:

    - **Send an image** — upload any portrait or headshot. The agent saves it as your avatar.
    - **Say "generate"** — the agent generates an AI avatar via OpenAI image models and asks if you want to keep it or regenerate.

    ```bash
    # The agent runs this when you say "generate":
    python pikastream-video-meeting/scripts/pikastreaming_videomeeting.py generate-avatar \
      --output identity/videomeeting-avatar.png
    ```
  </Step>

  <Step title="Set up your voice (first time only)">
    The agent also checks whether `life/voice_id.txt` exists. If it doesn't, it will ask:

    > I don't have a voice clone yet. You can: (a) send me a voice recording (10s-5min, clear speech) and I'll clone it, or (b) say "skip" to use a default voice.

    - **Send an audio recording** — the agent clones your voice and stores the voice ID locally.
    - **Say "skip"** — the agent uses the default voice (`English_radiant_girl`).

    ```bash
    # The agent runs this when you send an audio file:
    python pikastream-video-meeting/scripts/pikastreaming_videomeeting.py clone-voice \
      --audio your-recording.mp3 --name YourName --noise-reduction
    ```

    <Warning>
      Cloned voices may be deleted after 7 days of non-use. If your voice clone has expired, the agent will warn you and offer to re-clone from a new recording.
    </Warning>
  </Step>

  <Step title="Join a meeting">
    Drop a Google Meet or Zoom link into your agent chat. The agent automatically detects it and activates the `pikastream-video-meeting` skill.

    The agent will:
    1. Check your avatar and voice are ready
    2. Read your workspace context (MEMORY.md, identity files, recent activity)
    3. Build a concise system prompt for the bot
    4. Join the meeting as your AI avatar

    ```bash
    # The agent runs this join command:
    python pikastream-video-meeting/scripts/pikastreaming_videomeeting.py join \
      --meet-url https://meet.google.com/abc-defg-hij \
      --bot-name "Your Name" \
      --image identity/videomeeting-avatar.png \
      --system-prompt-file /tmp/meeting_system_prompt.txt \
      --voice-id your_voice_id_here
    ```

    Once joined, the agent will confirm: "I'm in the meeting."
  </Step>

  <Step title="Leave the meeting">
    To leave, tell your agent: `leave`. The agent runs the leave command with the session ID from the join step.

    ```bash
    python pikastream-video-meeting/scripts/pikastreaming_videomeeting.py leave \
      --session-id <session-id-from-join>
    ```

    After leaving, the agent retrieves and shares the meeting notes automatically.
  </Step>

  <Step title="Handle billing">
    Before every join, the skill checks your credit balance. If your balance is below the minimum threshold, it automatically creates a checkout link:

    ```json
    {
      "status": "needs_topup",
      "balance": 0,
      "product": "<credit pack name>",
      "credits": 500,
      "checkout_url": "https://...",
      "message": "Open the checkout URL to purchase <credit pack name>. Waiting for payment..."
    }
    ```

    Open the `checkout_url` in your browser to complete payment. The skill polls for payment automatically for up to 5 minutes. If payment completes within that window, the join proceeds automatically. If the timeout is exceeded, re-run the join command after paying.
  </Step>
</Steps>
